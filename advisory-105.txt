I was not going to write this up since it is a silly problem but this server is listed in the netcraft survey so people are using it. TheServer is a very small and simple webserver for the Windows platform it consists of a single executable and configuration file. 
TheServer stores the password for log file accesss in cleartext. This password is stored in the server.ini file. Which by default resides in the servers root directory. This is a VERY simple webserver for Windows 98/95 if the server is setup to log, then the password is also sent to the logfile when accessing the server logs remotely. The risk is 
you can have someone else parsing your weblogs that you never intended. 
</p><p>Hello Larry; 
</p><p>    I&amp;#039;m the developer of TheServer. I&amp;#039;ve posted information about the 
security issue you wrote about on Security Tracker on my website at 
<a href="http://www.fastlinksoftware.com">http://www.fastlinksoftware.com</a> under the FAQ section. The information I 
posted is as follows: <br/>
Q: How do I prevent people from downloading my SERVER.INI file and 
getting my system password? <br/>
A: Make a folder below the root of your web pages and place SERVER.EXE 
and SERVER.INI there. <br/>
Set the &amp;#039;root&amp;#039; field in the SERVER.INI file to the folder where your web 
pages start. <br/>
Add a &amp;#039;restrict&amp;#039; line for the new folder you created. 
Example: <br/>
Your web pages are in c:\web <br/>
Make a folder called &amp;#039;bin&amp;#039; under c:\web. 
Place SERVER.EXE and SERVER.INI in the &amp;#039;bin&amp;#039; folder. 
Edit the SERVER.INI file and change the &amp;#039;root&amp;#039; field as follows: 
root=c:\web <br/>
Add a line in the SERVER.INI file that would appear as follows: 
restrict=bin <br/>
With the above example when TheServer starts it will look for the 
starting web page in &amp;#039;c:\web&amp;#039; and access will be restricted to the &amp;#039;bin&amp;#039; 
folder that contains the SERVER.INI file.
