The distribution is focused on penetration testing and contains custom tools the SEA has developed.&lt;/span&gt;&lt;/td&gt;&lt;/tr&gt;&lt;tr&gt;&lt;td style="background-color: rgb(220,220,220);"&gt;&lt;b&gt;Vulnerability:&lt;/b&gt;&lt;div&gt;The tools appear to be developed in PHP &amp;amp; Perl and are web based.  These tools are served through a local apache instance running on the desktop system the apache webserver listening on <a href="telnet://0.0.0.0">0.0.0.0</a>:80 allowing remote access to all of the tools in their catalog. These tools allow remote command execution and file uploads, it was the intent of the SEA to have these only accessable to the local user.  The SEA is planning on fixing this vulnerability during their next release.
</p><p>A quick fix would be to modify /etc/apache2/ports.conf and bind apache to localhost:
</p><p># cat ports.conf |grep -n 127<br/>
8:NameVirtualHost <a href="telnet://127.0.0.1">127.0.0.1</a>:80<br/>
9:Listen <a href="telnet://127.0.0.1">127.0.0.1</a>:80
</p><p>This will prevent general network access to the scripts pre-loaded in the web root, but won&amp;#039;t help with Javascript payloads embedded in webpages like the PoC I provided below.
