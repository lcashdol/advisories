&lt;/span&gt;&lt;/td&gt;&lt;/tr&gt;&lt;tr&gt;&lt;td style="background-color: rgb(220,220,220);"&gt;&lt;b&gt;Vulnerability:&lt;/b&gt;&lt;div&gt;I noticed a /tmp race condition in IBM&amp;rsquo;s installation manager software install script
The code in consoleinst.sh is:
</p>
<p> 46 TEMP=/tmp<br/>
 47 tempScript=$TEMP/consoleinst-$$.sh<br/>
 48 scriptLoc=`dirname &amp;quot;$0&amp;quot;`<br/>
 49 slash=`expr &amp;quot;$scriptLoc&amp;quot; : &amp;quot;\(/\)&amp;quot;`
 50 if [ &amp;quot;X$slash&amp;quot; != &amp;quot;X/&amp;quot; ]; then
 51         scriptLoc=`pwd`/$scriptLoc<br/>
 52 fi<br/>
 53 <br/>
 54 if [ &amp;quot;$0&amp;quot; != &amp;quot;$tempScript&amp;quot; ]; then
</p><pre>
 55     cp &amp;quot;$0&amp;quot; &amp;quot;$tempScript&amp;quot;
 56     cd &amp;quot;$TEMP&amp;quot;
 57     origScriptLoc=$scriptLoc
 58     export origScriptLoc
 59     exec &amp;quot;$tempScript&amp;quot; $@
 60     # should not return from above exec
 61     exit 1
</pre>
<p> 62 fi
</p>
<p>If you guess the pid and create the file before the installer script does you can inject code to be executed at line 59.

[U] -rwxrwxrwx 1 larry larry 2225  Thu Oct 29 21:46:34 2015 /tmp/consoleinst-10382.sh
